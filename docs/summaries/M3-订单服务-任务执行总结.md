## 3. 核心设计要点

### 3.1 抢票下单全链路（PRD §8.1）
```
用户 → 网关(限流) → order(领令牌) → order(提交下单)
   → 幂等校验(消费令牌) → 防重锁(5s) → Lua预扣Redis(Feign)
   → 落单DB(Seata) → 本地消息表 → Kafka延迟消息(15min)
   → Kafka异步通知DB库存扣减 → 返回订单号+金额+截止时间
```

### 3.2 幂等性设计
- **下单令牌**：UUID 存 Redis（TTL 10min），提交时 Lua GETDEL 原子消费
- **防重锁**：Redis setNX 5 秒窗口，防止用户狂点
- **消费幂等**：Kafka 消费者使用 Redis setIfAbsent 去重

### 3.3 分库分表策略
- 分片键：`user_id`
- 路由算法：`user_id % 32` → 4 库 × 8 表
- 雪花 ID 低位嵌入 `user_id % 32`，保证同用户订单落在同一分片

### 3.4 分布式事务
- `@GlobalTransactional` 包裹下单核心流程（订单 + 明细 + 本地消息表）
- Seata AT 模式自动代理数据源

### 3.5 延迟取消机制
- 15 分钟超时取消（Kafka 延迟 Topic）
- 消费时二次校验订单状态（已支付则丢弃）
- 取消后异步归还 Redis + DB 库存
